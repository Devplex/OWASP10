Ziel:

	Dem Opfer intime Benutzerdaten stehlen.

Umgebung:

	Websites mit Textfelder als Eingabemöglichkeit.


Umsetzung:

	Der erste Schritt beinhaltet das finden der XSS Lücke innerhalb einer autoritären Website
	(z.B. www.google.de). Diese findet man, indem ein primitver HTML Code in ein Textfeld ein-
	gegeben wird. Wird dieser Code ausgeführt, ist die Lücke gegeben.

	Nun baut man mit einer Reihe von HTML Befehlen innerhalb dieser Website eine eigene Seite
	auf, welche auch mit eigenen Skripten erweitert werden kann. Ist die Seite fertig erstellt,
	sendet man diese z.B. via E-Mail seinen potentiellen Opfern zu und hofft, dass diese sich
	Täuschen lassen und Ihre intimen Zugangsdaten preisgeben.


Schutz:

	Benutzereingaben darf grundsätzlich nicht vertraut werden und somit dürfen diese niemals
	interpretiert werden. Sie bedürfen eine Validierung, welche Befehle erkennt und blockiert.

Unterschied Injection:

	Bei der Injection hat das Opfer nur eine passive Rolle. Bei XSS ist es dem Benutzer
	noch möglich einem Angriff zu entgehen indem er die Täuschung noch rechtzeitig bemerkt.
	Somit ist er hier aktiv involviert.
